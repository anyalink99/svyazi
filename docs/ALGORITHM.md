# Алгоритм семантических агентов

## 1. Пространство слов

Исходный Navec содержит 500 000 русских слов с 300-мерными векторами. Сборщик:

1. берёт частотные русские слова из `wordfreq`;
2. приводит их к нормальной форме через `pymorphy3`;
3. исключает служебные слова, имена, географию, организации, аббревиатуры и слова вне Navec;
4. выбирает 2 500 существительных для поля и 50 000 существительных, прилагательных, глаголов и наречий для подсказок;
5. L2-нормализует исходные векторы;
6. применяет воспроизводимую случайную ортогональную проекцию 300 → 128;
7. снова нормализует и квантует координаты в `int8`;
8. сохраняет по 192 ближайшие допустимые подсказки для каждого слова поля.

Для часто вызываемого поиска ведущего близости соседей записаны отдельно в `int16` **до** квантования. ИИ-игрок считает косинус между компактными `int8`-векторами:

```text
cos(a, b) = dot(q(a), q(b)) / (norm(q(a)) · norm(q(b)))
```

Так runtime-модель занимает около 11 МБ и загружается без Python.

## 2. Допустимость подсказки

Подсказка должна быть одним русским словом из семантического словаря. Для каждой карточки запрещаются:

- полное совпадение;
- совпадение Snowball-основы длиной от трёх символов;
- почти полный общий префикс;
- включение одного слова в другое для слов длиной от пяти символов.

Проверка намеренно строже минимальных правил настольной игры: агент не должен получать преимущество за счёт очевидной словоформы.

## 3. ИИ-ведущий

Пусть `T` — закрытые слова своей команды, а `B` — все закрытые карточки. Множество кандидатов — объединение ближайших соседей слов из `T`.

Для каждого кандидата `c` карточки из `B` сортируются по `cos(c, word)`. Безопасный префикс заканчивается перед первой карточкой другой роли. Рассматриваются размеры подсказки от 1 до `min(4, длина безопасного префикса)`.

Для размера `n` используются:

- `meanTarget` — средняя близость первых `n` своих слов;
- `weakestTarget` — близость самого слабого из них;
- `danger` — максимальная близость чужой карточки;
- `margin = weakestTarget - danger`;
- отдельные давления убийцы, противника и мирной карты.

Текущая функция:

```text
score = 2.15·n
      + 1.35·meanTarget
      + 0.80·weakestTarget
      + 5.20·margin
      - 2.70·assassinPressure
      - 0.80·enemyPressure
      - 0.35·neutralPressure
```

Уверенность показывается как сигмоида запаса:

```text
confidence = sigmoid(12 · (margin - 0.015))
```

Если среди первых 56 соседей безопасного слова нет, поиск расширяется до всех 192. В исключительно редкой позиции, где ни одна допустимая подсказка не ставит своего агента первым, выбирается контролируемый рискованный вариант на одно слово. Он минимизирует число чужих карт перед целью и получает особенно большой штраф, если первой окажется карта убийцы. Это гарантирует продолжение партии и моделирует вынужденный человеческий риск.

Контекст всех 25 слов, таблицы соседей и результаты морфологической проверки кешируются для нескольких последних полей.

## 4. ИИ-игрок

Игрок не получает роли. Для каждой закрытой карточки он вычисляет:

```text
perceivedScore = cosineSimilarity + Normal(0, profileNoise)
```

Шум детерминирован seed партии, подсказкой, числом и профилем — одинаковый эксперимент всегда даёт одинаковый результат.

| Профиль | σ шума | Минимальная связь | Лишняя попытка |
| --- | ---: | ---: | --- |
| осторожный | 0.018 | 0.24 | нет |
| сбалансированный | 0.038 | 0.12 | нет |
| азартный | 0.068 | 0.02 | да |

Осторожный и сбалансированный игроки также останавливаются, когда следующие варианты слабы и почти неразличимы.

## 5. Self-play

Симулятор создаёт поле из одного seed, по очереди вызывает ведущего и игрока, применяет обычные правила открытия и собирает агрегаты. Он не делает сетевых запросов, поэтому подходит для:

- сравнения эмбеддингов;
- настройки коэффициентов риска;
- сравнения профилей игроков;
- поиска редких аварийных позиций;
- регрессионных прогонов после изменения словаря.

Для корректного сравнения моделей нужно фиксировать seed, число партий, размер соседей и профили обеих команд.
